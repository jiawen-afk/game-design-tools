# 图片处理工作台逐图参数设计

## 目标

图片批次中的每张图片都保留自己的完整处理参数。切换图片只切换编辑上下文，不重置参数；只有用户点击“全部应用预览”时，才把当前图片的参数同步到全部图片。

## 行为约定

- 新上传的每张图片获得一份独立默认配置，裁剪范围默认为整张图片。
- 编辑当前图片时，抠图开关、抠图模式和参数、裁剪框、导出编码、导出背景、导出缩放、高清化开关和高清化参数都写回当前批次项。
- 切换图片前保存当前配置，切换后恢复目标图片配置；预览对象可以清理，但配置不得重置。
- 点击“全部应用预览”时，复制当前图片的完整配置到全部批次项。裁剪框依据源图和目标图尺寸按比例映射。
- 批量导出和批量高清化逐项读取各自配置。不同图片允许使用不同编码格式、背景、缩放和处理参数。
- 高清化预览清理只在确实保存过导出缩放快照时恢复该快照，不能用默认值 1 覆盖目标图片配置。

## 架构

在纯模型中定义 `ImageProcessingBatchSettings`，负责默认值、深拷贝、按尺寸映射和逐项文件名派生。`ImageProcessingBatchItem` 持有自己的 `settings`。工作台组合 hook 负责把当前面板状态捕获成配置，并把目标配置恢复到各子 hook；图片来源 hook 只负责批次资源生命周期和切换编排；导出 workflow 逐项消费配置，不再把当前图片的一组全局参数套用到所有图片。

## 数据流

1. 上传：创建图片草稿和默认配置。
2. 编辑：当前 UI 状态变化时更新活动批次项的配置。
3. 切换：保存活动配置，清理易失预览，加载目标草稿和目标配置。
4. 全部应用：复制活动配置到所有项，对裁剪框做比例映射；启用高清化时再逐项生成预览。
5. 导出：每个批次项用自己的配置完成抠图、裁剪、缩放、背景合成、高清化和编码。

## 错误与资源处理

- 预览失败不修改已经保存的逐图配置。
- 批量处理中任一项失败，沿用现有错误提示并清理本次生成的临时 URL。
- 切换图片会撤销当前易失预览 URL，但不会销毁批次图片草稿或配置。

## 测试

- 纯模型测试覆盖默认配置、独立深拷贝、逐图恢复、全部应用和比例裁剪。
- 结构测试覆盖高清化预览清理不再无条件重置缩放，以及工作台通过聚焦的逐图配置边界编排状态。
- 工作流测试覆盖逐项导出文件扩展名和批量预览签名。
- 完整运行 `npm test`、`npm run build` 和 `git diff --check`。

## 自检

- 无占位内容。
- “逐图独立”与“全部应用后一致”由显式操作区分，不存在隐式同步。
- 裁剪框对不同尺寸图片采用比例坐标，其他参数直接深拷贝。
- 本设计只处理图片处理工作台的批次配置，不扩展到精灵图工作台。
