# v0.7.0 README, Third-Party Notices, and Release Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Publish `v0.7.0` with synchronized English and Chinese documentation plus an accurate, categorized in-app list of major third-party runtimes, models, direct libraries, and build tools.

**Architecture:** Keep third-party metadata as typed static data in `src/openSourceSoftware.ts`; let `SiteFooter` only group and render that data. Protect the data and rendering contract with the existing app-shell structure test, keep README content human-maintained and bilingual, and use the repository's GitHub Actions workflow for the Windows release.

**Tech Stack:** TypeScript, React 19, Ant Design 6, Electron, Node test runner, Vite, GitHub Actions, GitHub CLI.

## Global Constraints

- Work directly on `master` as explicitly requested.
- Publish version `0.7.0` and annotated tag `v0.7.0`.
- List major runtimes, AI models/services, direct production libraries, and key build tools; do not list every transitive dependency.
- Display project links and every applicable license link in the About dialog.
- Distinguish open-source code licenses from model-weight licenses and additional model terms.
- Describe the pinned video runtime as an LGPL shared FFmpeg build and do not claim H.265 is universally royalty-free.
- Keep `README.md` and `README.zh-CN.md` structurally aligned.
- Stop the release if tests, build, push, workflow, or release verification fails.

```ts
export interface OpenSourceLicense {
  name: string
  url: string
}

export type OpenSourceSoftwareCategory =
  | 'desktop-media'
  | 'ai-models'
  | 'application-libraries'
  | 'build-tools'

export interface OpenSourceSoftwareItem {
  category: OpenSourceSoftwareCategory
  name: string
  usage: string
  url: string
  licenses: OpenSourceLicense[]
}

export const openSourceSoftwareCategories: Array<{
  id: OpenSourceSoftwareCategory
  label: string
}> = [
  { id: 'desktop-media', label: '桌面与媒体运行时' },
```

```ts
]
```

- [ ] **Step 4: Render categories, project pages, and licenses**

```tsx
import { openSourceSoftware, openSourceSoftwareCategories } from '../openSourceSoftware'
```

```tsx
<p className="open-source-intro">
  以下列出本应用使用的主要第三方运行时、模型、直接依赖和构建工具；完整 npm 依赖请查阅 package.json 与 package-lock.json。

    <section className="open-source-category" key={category.id}>
      <h3>{category.label}</h3>
      <div className="open-source-category-items">
        {openSourceSoftware
          .filter((item) => item.category === category.id)
          .map((item) => (
            <article className="open-source-item" key={item.name}>
              <a className="open-source-project" href={item.url} target="_blank" rel="noreferrer">
                <strong>{item.name}</strong>
              </a>
              <span>{item.usage}</span>
              <span className="open-source-licenses">
                许可证：
                {item.licenses.map((license, index) => (
                  <a
                    className="open-source-license"
                    href={license.url}
                    key={`${item.name}-${license.name}`}
                    target="_blank"
                    rel="noreferrer"
                  >
                    {index > 0 ? `；${license.name}` : license.name}
                  </a>
                ))}
              </span>
            </article>
          ))}
      </div>
    </section>
  ))}
</div>
```

Set the About modal width to `760`, give its body a maximum viewport height, and add focused CSS classes for the intro, category headings, item cards, project links, and license links. Preserve visible keyboard focus and underline links on hover.
